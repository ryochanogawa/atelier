# フロントエンド知識

## コンポーネント設計

1つのファイルにすべてを書かない。常にコンポーネントを分割する。

分割が必要な場合:
- 独自の状態を持つ → 分割必須
- JSX が50行超 → 分割
- 再利用可能 → 分割
- 複数の責務 → 分割
- ページ内の独立したセクション → 分割

| 基準 | 判定 |
|------|------|
| コンポーネント 200行超 | 分割を検討 |
| コンポーネント 300行超 | REJECT |
| 表示とロジックが混在 | 分離を検討 |
| Props の受け渡し（3階層以上） | 状態管理を検討 |
| 複数の責務を持つコンポーネント | REJECT |

良いコンポーネント:
- 単一責務: 1つのことをうまくやる
- 自己完結: 依存が明確
- テスト可能: 副作用が分離されている

コンポーネント分類:

| タイプ | 責務 | 例 |
|--------|------|-----|
| Container | データ取得、状態管理 | `UserListContainer` |
| Presentational | 表示のみ | `UserCard` |
| Layout | 配置、構造 | `PageLayout`, `Grid` |
| Utility | 共通機能 | `ErrorBoundary`, `Portal` |

ディレクトリ構造:
```
features/{機能名}/
├── components/
│   ├── {機能}-view.tsx      # メインビュー（子を合成）
│   ├── {サブコンポーネント}.tsx
│   └── index.ts
├── hooks/
├── types.ts
└── index.ts
```

## ルーティング接続（ページ追加時）

ページコンポーネントの作成だけで終わらない。新しいページは実際のエントリパスにも接続する必要がある。ルーター、メニュー、一時ルートなどのエントリポイントをページ実装と同時に決定する。

| 基準 | 判定 |
|------|------|
| 新しいページがあるがルートが登録されていない | REJECT |
| ベースネームベースのURLとルートパスの対応が検証されていない | REJECT |
| ルーター接続とページエントリがページ実装と同時に決定 | OK |
| 開発用一時ルートの目的と削除計画が記録 | OK |
| ルートは更新されたがメニュー、ボタン等の実際のエントリポイントが未確認 | Warning |

到達可能性はルーター設定よりも広い。ユーザーが実際にたどるエントリパス（メニュー、遷移ボタン、ダイアログアクション、他画面からのリンク、外部呼び出し元）を確認する。

### サードパーティUIライブラリの統合

データグリッド、日付ピッカー、チャートなどのサードパーティUIライブラリは、型が通っていてもランタイムで失敗する可能性がある。特にメジャーバージョン変更時に props 名や状態モデルの形状が変わっている場合、浅いモックでは問題を検出できない。

| 基準 | 判定 |
|------|------|
| プロジェクトのバージョンを確認せずにUI ライブラリの props を推測 | REJECT |
| テストがライブラリを完全にモック化して実際のマウント失敗を見逃す | Warning |
| 代表的な props で実コンポーネントをレンダリングし画面レベルでクラッシュしないことを検証 | OK |
| 既存のプロジェクト内の使用パターンとインストール済みバージョンを参照して props 形状を選択 | OK |

## 状態管理

子コンポーネントは自身の状態を変更しない。イベントを親にバブルアップし、親が状態を操作する。

```tsx
// NG - 子が自身の状態を変更
const ChildBad = ({ initialValue }: { initialValue: string }) => {
  const [value, setValue] = useState(initialValue)
  return <input value={value} onChange={e => setValue(e.target.value)} />
}

// OK - 親が状態を管理、子はコールバックで通知
const ChildGood = ({ value, onChange }: { value: string; onChange: (v: string) => void }) => {
  return <input value={value} onChange={e => onChange(e.target.value)} />
}
```

例外（子がローカル状態を持ってよい場合）:
- UIのみの一時的な状態（ホバー、フォーカス、アニメーション）
- 親に伝達する必要のない完全にローカルな状態

| 基準 | 判定 |
|------|------|
| 不要なグローバル状態 | ローカル化を検討 |
| 同じ状態が複数箇所で管理 | 正規化が必要 |
| 子から親への状態変更（逆方向のデータフロー） | REJECT |
| APIレスポンスをそのまま状態に格納 | 正規化を検討 |
| 不適切な useEffect 依存配列 | REJECT |
| 不安定な Context/Provider 関数参照に初期ロードが紐づく | REJECT |

状態の配置ガイドライン:

| 状態の性質 | 推奨配置 |
|-----------|---------|
| 一時的なUI状態（モーダルの開閉等） | Local (useState) |
| フォーム入力値 | Local またはフォームライブラリ |
| 複数コンポーネントで共有 | Context または状態管理ライブラリ |
| サーバーデータキャッシュ | データフェッチライブラリ（TanStack Query 等） |

## 初期ロードとリフェッチの境界

初期ロードはリアクティブなリフェッチから分離する。リフェッチが URL、フィルタ、ページング、明示的なユーザーアクションに駆動されない場合は、マウント時のみとし、不安定なコールバック参照に紐づけない。

| 基準 | 判定 |
|------|------|
| Provider/Context コールバックの ID 変更で初期ロードが再実行 | REJECT |
| リフェッチ条件が明示的（URL、フィルタ、ページング、更新アクション） | OK |
| メッセージ表示、ローディングトグル、モーダル状態がリフェッチを引き起こす | REJECT |
| 初期ロードがマウント時のみで、後続のリフェッチが明示的にトリガー | OK |

## データフェッチ

API 呼び出しはルート（View）コンポーネントで行い、props 経由で子に渡す。

```tsx
// OK - ルートで取得、子に渡す
const OrderDetailView = () => {
  const { data: order, isLoading, error } = useGetOrder(orderId)
  const { data: items } = useListOrderItems(orderId)

  if (isLoading) return <Skeleton />
  if (error) return <ErrorDisplay error={error} />

  return <OrderSummary order={order} items={items} />
}

// NG - 子が自分でデータを取得
const OrderSummary = ({ orderId }) => {
  const { data: order } = useGetOrder(orderId)
  // ...
}
```

UI 状態の変更がパラメータに影響する場合（週切り替え、フィルタ等）は View レベルで状態を管理し、コールバックをコンポーネントに渡す。

例外（コンポーネントレベルのフェッチが許可されるケース）:

| ケース | 理由 |
|--------|------|
| 無限スクロール | スクロール位置（内部UI状態）に依存 |
| オートコンプリート | ユーザー入力に基づくインクリメンタル検索 |
| 遅延ロード | 表示時のみ必要なデータ |

## アクセシビリティ

### 基本要件

| 基準 | 判定 |
|------|------|
| 画像に alt 属性がない | REJECT |
| フォーム要素に label が関連づけられていない | REJECT |
| カラーのみで情報を伝達 | REJECT |
| キーボードでフォーカスできないインタラクティブ要素 | REJECT |
| 適切な見出し階層がない | Warning |
| ARIA 属性の不適切な使用 | REJECT |

### セマンティックHTML

| 要素 | 用途 |
|------|------|
| `<button>` | クリック可能なアクション |
| `<a>` | ナビゲーション |
| `<nav>` | ナビゲーションのグループ |
| `<main>` | メインコンテンツ |
| `<article>` | 自己完結した内容 |
| `<section>` | テーマ的なグループ |

```tsx
// NG - div をボタンとして使用
<div onClick={handleClick}>送信</div>

// OK - セマンティックなボタン
<button type="button" onClick={handleClick}>送信</button>
```

### フォーカス管理

- モーダルが開いたらフォーカスをモーダル内に移動
- モーダルが閉じたらフォーカスをトリガー要素に戻す
- フォーカストラップの実装（Tab でモーダル外に出ない）
- skip navigation リンクの提供
